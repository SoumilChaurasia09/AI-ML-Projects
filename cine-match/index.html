Mathematical Formulation & RAG Explainer

⚡ 1. Cosine Similarity Vector Search

Relevance between a user's prompt query vector \(Q\) and a movie document vector \(D\) is computed as the cosine of the angle between them. Since vectors are normalized to unit length (\(\|Q\| = \|D\| = 1\)), the formula simplifies to a dot product:

CosineSim(Q, D) = Q • D = Σ (q_i * d_i)

Where \(q_i\) is the TF-IDF weight of word \(i\) in the search query, and \(d_i\) is the TF-IDF weight of the same word in the movie description.

⚡ 2. User Taste Profiling

We build a user preference vector \(U\) by aggregating vectors of liked movies and subtracting disliked movie vectors.

U = (1/N) * Σ D_liked - (0.3/M) * Σ D_disliked

We then rank unviewed movies in the catalog by computing \(CosineSim(U, D_{unviewed})\). The dashboard updates your "Recommended for You" row in real-time as you click Like/Dislike.
